# agentscore — Product Requirements Document

## 1. 문제 정의

### 배경

Claude Code 사용 환경이 복잡해지고 있다.
- 2023: Claude Code 단독
- 2024: MCP 서버 2~3개 추가
- 2026: bkit + oh-my-claudecode + 각종 MCP + 스킬 수십 개

이 복잡도는 Kubernetes 클러스터 수준에 도달했지만, `kubectl describe`에 해당하는 진단 도구가 없다.

### 핵심 문제 (1문장)

> AI 코딩 에이전트(Claude Code) 사용자는 자신의 도구 환경(MCP 서버, 플러그인, 스킬, 권한 설정)이 얼마나 잘 구성됐는지 객관적으로 평가할 수단이 없어, 잠재적 기능 격차와 잘못된 설정을 인식하지 못한 채 사용한다.

### 구체적 증상

| 문제 | 설명 |
|------|------|
| **가시성 없음** | 내 설정이 잘 됐는지 알 방법이 없음 |
| **격차 모름** | 중요한 도구가 빠진 것을 인식하지 못함 |
| **충돌 모름** | 20개 플러그인 중 3개가 충돌하는지 알 수 없음 |
| **기준 없음** | '잘 된 환경'의 정의 자체가 없음 |
| **컨텍스트 낭비** | 어떤 도구가 컨텍스트를 불필요하게 소비하는지 모름 |
| **도구 평가 어려움** | 새 GitHub 프로젝트를 설치해야 할지 매번 수동으로 판단 |

### 현재 해결 방법과 한계

| 현재 방법 | 한계 |
|----------|------|
| 문서 직접 읽기 | 시간 비용 과다, 비구조적 |
| 커뮤니티 질문 | 재현 불가, 주관적 |
| 남의 설정 복사 | 이유 모름, 맹목적 의존 |
| 시행착오 | 설치 후에야 문제 발견 |

### 기존 도구와의 차별점

| 도구 | 평가 대상 | 한계 |
|------|----------|------|
| Factory.ai Agent Readiness | 코드 리포지토리 | 도구 환경이 아님 |
| kodustech/agent-readiness | 코드 리포지토리 | 동일 |
| cc-plugin-eval | 플러그인 트리거 | 종합 점수/권장 없음 |
| Lighthouse | 웹 성능 | AI 환경 무관 |
| **agentscore** | **AI 개발 도구 환경 자체** | — |

## 2. 제품 개요

### 한 줄 설명

> "Lighthouse for AI agent development environments"

### 핵심 원칙

1. **자기 모순 없음**: agentscore 자체가 Claude Code 세션에 컨텍스트 0 추가
2. **On-demand 실행**: 평소엔 완전 침묵, 필요할 때만 실행
3. **즉각 가치**: 설치 후 30초 안에 첫 결과
4. **상태 지속**: 히스토리 자동 저장, 트렌드 추적
5. **경량 저장소**: 30일치 < 500KB, 외부 서버 없음
6. **점진적 정밀도**: 프로필 없이도 유용, 상세할수록 더 정확

## 3. 사용자

### 주요 타겟

- Claude Code를 플러그인/MCP와 함께 사용하는 개발자 (3개 이상 설치)
- 팀 내 AI 도구 환경을 표준화하려는 개발자/팀 리드
- 새로운 AI 도구를 자주 탐색하는 얼리어답터

### 사용 시나리오

1. 처음 설정 후 "내 환경이 잘 됐나?" 확인
2. 새 플러그인 설치 전 "이거 설치해도 될까?" 판단
3. 환경이 느려진 것 같을 때 "뭐가 문제인지" 진단
4. 팀원의 환경과 내 환경을 동일하게 맞추고 싶을 때

## 5. 평가 프레임워크

### 6개 평가 차원

| 차원 | 배점 | 측정 방식 |
|------|------|----------|
| Context Efficiency | 25점 | 혼합 (정적 + 동적) |
| Coverage | 20점 | 정적 (프로필 기반) |
| Conflict Detection | 20점 | 정적 + 동적 |
| Configuration Quality | 15점 | 정적 |
| Security | 10점 | 정적 |
| Freshness | 10점 | 정적 |

### Context ROI 개념 (핵심 차별점)

컨텍스트 비용을 절대값이 아닌 **가치 대비 비율**로 측정한다.

```
Context ROI = (프로필 적합성 × 설정 우선순위) / 컨텍스트 비용
```

**"사용 빈도" 대신 "설정 우선순위 신호"를 사용한다.** 로그 파싱 없이 설정 파일만으로 신뢰할 수 있는 신호를 추출한다.

| 신호 | 설정 우선순위 |
|------|------------|
| CLAUDE.md에서 직접 언급됨 | 높음 |
| `settings.json → enabledPlugins` 활성화 | 중간 |
| `settings.local.json → enabledMcpjsonServers` 등록 | 중간 |
| `mcp.json → mcpServers` 설정 | 중간 |
| 설치만 됐고 어디에도 없음 | 낮음 |

| 케이스 | 판정 |
|--------|------|
| 비용 높음 + 프로필 적합 + 우선순위 높음 | 정당한 비용 (감점 없음) |
| 비용 높음 + 프로필 부적합 + 우선순위 낮음 | Dead Weight (강한 감점) |
| 비용 낮음 + 프로필 적합 | 이상적 (보너스) |

**컨텍스트 비용 표기:**
- registry 등록 도구: 하드코딩된 값 사용
- 미등록 도구: 구조 분석 기반 추정값 + "(estimated)" 표시
- `--measure` 플래그: 실제 Claude 실행으로 측정 후 "(measured)" 표시

### 점수 등급

| 등급 | 점수 | 설명 |
|------|------|------|
| S | 90-100 | 최적화된 환경 |
| A | 75-89 | 잘 설정됨, 소규모 개선 필요 |
| B | 55-74 | 기본은 갖춰짐, 중요 개선 필요 |
| C | 35-54 | 상당한 격차 존재 |
| D | 0-34 | 재구성 권장 |

## 6. 프로필 시스템

### 3단계 경험 (Progressive Disclosure)

```
Tier 0: 즉시 실행 — 범용 점수 출력, 프로필 선택 제안
Tier 1: 역할 선택 — backend / frontend / fullstack / ml / devops
Tier 2: 상세 설정 — 스택, 작업 패턴, 팀 규모, 우선순위
```

### 프로필 자동 감지

실행 시 현재 디렉토리와 설치 도구를 분석해 프로필을 추정한다.

- `.py` 파일 다수 + no `package.json` → backend / ml 후보
- `package.json` + tailwind → frontend 후보
- `Dockerfile` + k8s yaml → devops 후보

### 프로필이 점수에 미치는 영향

동일한 도구라도 프로필에 따라 가치 평가가 달라진다.

예시: `bkit:enterprise`
- devops 프로필 → Coverage +3 (필수 도구)
- frontend 프로필 → Conflict -2 (컨텍스트 낭비)

## 7. `agentscore check` 상세

### 목적

새 GitHub 프로젝트를 현재 환경에 추가할 때 **설치 전** 영향을 시뮬레이션한다.

### 분석 항목

1. 도구 유형 분류 (MCP 서버 / 플러그인 / 스킬 / 라이브러리)
2. 컨텍스트 비용 추정 (manifest, README 분석)
3. 기존 도구와 기능 중복 검사
4. 프로필 적합성 평가
5. 점수 시뮬레이션 (설치 전/후 예측)
6. GitHub 신뢰도 지표 (stars, 최근 커밋, 라이선스)

### 출력

- 설치 권장 / 비권장 / 조건부 권장
- 점수 변화 예측 (+/- 분해)
- 최적 시나리오 제안 (예: "기존 X 제거 후 설치 시 +3점")

## 8. MVP 범위

### MVP에 포함

- `agentscore` (범용 즉시 스캔)
- 6개 평가 차원 기본 구현
- Tier 0 + Tier 1 프로필 (5개 역할)
- Context ROI 기본 계산
- `agentscore check <github-url>`
- 히스토리 저장 (JSON)

### v2로 미루는 것

- `agentscore watch` (파일 감시)
- `agentscore compare` (배치 비교)
- 동적 평가 심화 (실제 실행 측정)
- Tier 2 상세 프로필
- 팀 공유 기능
- 다른 AI 코딩 도구 지원 (Claude Code 전용 → 확장)
